## üìö Key Learnings

### 1. Ethical Hacking vs. Bug Bounty
* **Ethical Hacking:** A broad field involving authorized probing of systems to fix security flaws.
* **Bug Bounty Hunting:** A crowdsourced model where hackers are paid "per vulnerability" found (e.g., via HackerOne).
* **Penetration Testing:** A professional service where a company hires a hacker for a specific scope and time (Pay-per-service).

### 2. Common Myths
| Myth ‚ùå | Reality ‚úÖ |

### 3. The 5 Phases of Ethical Hacking
1.  **Reconnaissance:** Gathering info (IPs, Domains) passively.
2.  **Scanning:** Active probing for open ports and services.
3.  **Gaining Access:** Exploiting vulnerabilities to enter the system.
4.  **Maintaining Access:** Ensuring re-entry (often skipped in white-hat scenarios).
5.  **Clearing Tracks / Reporting:** Removing logs or, more importantly for ethical hackers, documenting the findings.

### 4. Types of Penetration Testing
* ‚¨õ **Black Box:** No prior knowledge of the target. Simulates an external attack.
* ‚¨ú **White Box:** Full access to source code and network maps.
* Yz **Gray Box:** Partial access (e.g., user-level credentials).

## üåê Networking Fundamentals

### Network Types (by Scale)
* **PAN:** Personal Area Network (Bluetooth).
* **LAN:** Local Area Network (Home/Office Wi-Fi).
* **MAN:** Metropolitan Area Network (City-wide).
* **WAN:** Wide Area Network (The Internet).

### The Client-Server Model
* **Client:** The requester (e.g., Browser).
* **Server:** The provider (e.g., Google's Data Center).
* **Communication:** Request ‚û°Ô∏è Response.

### Data Packets üì¶
Data is broken into small chunks called **packets** to travel across the net.
* **Header:** Source & Destination IP.
* **Payload:** The actual data.
* **Trailer:** Error checking.

### Types of Servers
* **Web Server:** Hosts websites (HTTP/HTTPS).
* **DNS Server:** Translates URLs to IP addresses.
* **Mail Server:** Manages email traffic.
* **FTP Server:** For file transfers.
